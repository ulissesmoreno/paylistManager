# Regras do Framework Get Shit Done

1. **Sempre** leia o `STATE.md` antes de qualquer ação.
2. **Commits Atômicos:** Um commit para cada tarefa (T-task) do ROADMAP.
3. **Verificação de Código:** Compilar o Java após cada alteração estrutural.
4. **Memória:** Ao finalizar uma tarefa, atualize o `STATE.md` com o progresso e o próximo passo técnico.
5. **Clean Code:** Use nomes em inglês e siga os princípios SOLID estritamente.
